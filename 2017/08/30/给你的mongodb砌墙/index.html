为你的mongodb砌堵墙吧，防止被攻击和勒索

跟着这篇博客 我开始了操作。下面罗列一下砌墙碰到的问题：

1.linux下设置了一个用户,却无法访问

通过db.addUser(“admin”, “admin”)创建的用户
但是验证的db.auth(“admin”,”admin”)时候返回0

用mongo 127.0.0.1:27017/admin -u admin -p admin 也提示login失败
提示：

Failed to authenticate admin@admin with mechanism MONGODB-CR: AuthenticationFailed: UserNotFound: Could not find user admin@admin

查看了下linux上都是2.0.6的shell，db是3.0.6的.感觉可能会是版本的问题。
试着在mac上试了一下，完全没问题，mac上是最新版本 mongodb,MongoDB shell version v3.4.7。
后面我去下载了最新的linux mongodb包就可以了。
（由于linux比较生疏，所以操作比较蹩脚，花了挺多时间。）

2.加了权限之后，nodejs连接方式就要修改了

根据官方的格式去修改

mongodb://[username:password@]host1[:port1][,host2[:port2],…[,hostN[:portN]]][/[database][?options]]

由原来的 var DB_CONN_STR = ‘mongodb://106.14.151.155:27017/blog’;
改为 var DB_CONN_STR = ‘mongodb://blog:123@106.14.151.155:27017/blog’;

@前的 blog是账号，123是密码
